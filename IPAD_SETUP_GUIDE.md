## 一、电脑端准备（只需做一次）

在开始之前，你需要在电脑上准备好以下工具：

### 1. 安装 Node.js

Node.js 是用来运行部署脚本的环境。

1. 打开浏览器，访问 **https://nodejs.org**
2. 下载 **LTS 版本**（推荐，长期支持版）
3. 双击安装包，一路点 **"Next"（下一步）** 完成安装
4. 安装完成后，验证一下（见下方 Windows/Mac 章节）

### 2. 解压项目 ZIP

把收到的 ZIP 文件解压到一个文件夹，记住文件夹位置。

### 3. 用命令行打开项目目录

- **Windows：** 在项目文件夹里按住 Shift + 右键 → "在此处打开 PowerShell 窗口"
- **Mac：** 打开 "终端"（Terminal），输入 `cd ` 然后把项目文件夹拖进去，回车

### 4. 部署平台

本项目使用 **GitHub Pages** 部署，自动提供 HTTPS（PWA 必需）。

部署只需推送代码即可，无需额外安装 CLI 工具。

## 二、Windows 操作步骤

> 以下命令在 **PowerShell** 中执行（不是 CMD 命令提示符）。

### Step 1: 检查 Node.js

```powershell
node -v
```

应该显示版本号，例如 `v20.10.0`。如果没有，请重新安装 Node.js。

### Step 2: 进入项目目录

```powershell
cd "你的项目文件夹路径"
```

例如：`cd "C:\Users\你的用户名\Desktop\打卡"`

### Step 3: 运行 PWA 检查

```powershell
node scripts/check-pwa.js
```

应该看到 `🎉 PWA 配置完全通过，可以部署！` 如果没有通过，先修复报错再继续。

### Step 4: 更新缓存版本号

```powershell
node scripts/bump-cache-version.js
```

这会自动更新版本号，确保 iPad 上线后能拿到最新代码。

### Step 5: 部署

```powershell
git add -A; git commit -m "更新部署"; git push
```

推送后约 1-2 分钟自动部署完成。

部署网址：
```
https://clairechen0088-sketch.github.io/daka-pwa/
```

**🔴 把这个网址记下来，iPad 上要用！**

## 四、iPad 安装步骤

> ⚠️ **一定要用 Safari！** 不能用微信、Chrome、或其他浏览器。

### 1. 打开 Safari

找到 iPad 上的 **Safari** 图标（像指南针 🧭），点开它。

### 2. 输入网址

1. 点击屏幕顶部的地址栏
2. 输入 `https://clairechen0088-sketch.github.io/daka-pwa/`
3. 点击键盘上的 **"前往"**（Go）

> 💡 **小技巧：** 如果网址太长，可以先用电脑把网址通过微信/QQ 发给自己，然后在 iPad 上复制粘贴到 Safari。

### 3. 确认页面正常打开

你应该能看到：
- 页面显示了 "今日挑战"
- 底部有四个导航按钮（今日挑战 / 奖励 / 成长 / 家长）
- 页面加载完整，没有报错

### 4. 添加到主屏幕

这是**最关键**的一步：

1. 点击 Safari 地址栏右侧（或底部）的 **分享按钮**
   - 图标：一个方框里面有个向上的箭头 ↗️（新版 iPadOS 在地址栏右侧）
   - 旧版 iPad 在底部工具栏中间

2. 在弹出的菜单中，**向下滑动**

3. 找到并点击 **"添加到主屏幕"**（Add to Home Screen）
   - 图标：一个加号 ➕ 在方框上

4. 你会看到一个预览页面：
   - 名称应该显示为 **"打卡积分"**
   - 图标应该能看到

5. **点击右上角的 "添加"**

### 5. 从主屏幕打开

1. 回到 iPad 桌面（按 Home 键 或 从底部往上滑）
2. 找到 **"打卡积分"** 图标
3. 点击图标

### 6. 确认全屏模式

打开后确认：
- ✅ **没有 Safari 地址栏**（顶部没有网址输入框）
- ✅ **没有底部工具栏**（没有前进/后退/分享按钮）
- ✅ 看起来就像一个真正的 App

> 🎉 **恭喜！打卡积分系统已经像真正的 App 一样在 iPad 上运行了！**

## 六、每次更新代码后的操作

当你收到"有新版本"的通知后，按以下步骤操作：

### 电脑端

1. **先更新缓存版本号**（非常重要！）

```powershell
# Windows PowerShell
node scripts/bump-cache-version.js
```

```bash
# Mac 终端
node scripts/bump-cache-version.js
```

2. **提交并推送**

```powershell
# Windows / Mac 通用
git add -A; git commit -m "更新部署"; git push
```

### iPad 端

1. 从桌面打开 "打卡积分" App
2. **等待 1-2 秒**，系统会自动检测新版本并刷新
3. 如果还是旧版本：
   - 双击 Home 键（或从底部上滑停留），找到"打卡积分"
   - **向上滑完全关闭** App
   - 重新从桌面打开
4. 如果仍然是旧版本（极少见），等待 5 分钟后重试（Service Worker 更新需要时间）

> ⚠️ **不要随便清除 Safari 网站数据！** 详见下方数据安全提醒。
>
> ⚠️ **不需要重新"添加到主屏幕"！** 更新部署后，已安装的主屏幕 App 会自动获取新版本。

## 七、数据安全提醒 ⚠️

### 重要！请仔细阅读！

这个打卡系统使用的是 **iPad 本地存储（IndexedDB）**，数据存在你 iPad 的 Safari / PWA 环境里，**不经过任何服务器**。

| ⚠️ 危险操作 | 后果 | 怎么办 |
|------------|------|--------|
| 删除 Safari 网站数据 | **所有打卡记录、积分、兑换记录全部丢失！** | 不要随便清除！ |
| 卸载主屏幕 App | **数据全部丢失** | 不要卸载！ |
| 清除 Safari 历史记录和网站数据 | **数据可能被清空** | 只清除历史记录，不要勾选"网站数据" |
| iPad 恢复出厂设置 | **数据全部丢失** | 提前导出备份 |

### ✅ 安全操作

| 操作 | 是否安全 | 说明 |
|------|----------|------|
| 正常使用 App | ✅ 安全 | 数据保存在本地 IndexedDB |
| 部署更新 | ✅ 安全 | 更新不会清空数据 |
| iPad 系统更新 | ✅ 安全 | iOS 升级不影响 IndexedDB |
| App 闪退/重启 | ✅ 安全 | 数据自动持久化 |
| 导出完整备份 | ✅ 推荐 | 在家长端 → 设置 → 导出完整备份 |

### 📤 如何备份数据

1. 在 iPad 上打开 "打卡积分" App
2. 点击底部 **"家长"** → 输入 PIN 码
3. 进入 **"设置"**
4. 点击 **"📤 导出完整备份"**
5. 会下载一个 `.json` 文件
6. 将这个文件保存到电脑或云盘

> 💡 **建议：每周导出一次完整备份，** 把备份文件存在电脑上。万一数据出问题，可以导入恢复。

### 📋 备份规则（重要！请遵守）

| 规则 | 说明 |
|------|------|
| 🗓️ **每周手动导出一次完整备份** | 建议每周末固定导出一次，设置页会显示上次备份时间，超过 7 天会有提醒 |
| 🔄 **每次代码更新前先导出完整备份** | 部署新版本前务必先备份，以防兼容性问题 |
| ⬇️ **每次完整导入前先导出当前备份** | 导入旧备份会覆盖当前所有数据，先导出当前数据避免误操作丢失 |
| ❌ **不要随便清除 Safari 网站数据** | 清除网站数据会删除所有打卡记录、积分、兑换记录 |
| ❌ **不要随便卸载主屏幕 App** | 卸载会同时清除 IndexedDB 数据 |
| ⚠️ **清除网站数据前必须先导出完整备份** | 如果确实需要清除 Safari 数据，务必先导出并确认备份文件已保存 |
| ☁️ **JSON 备份文件建议保存到 iCloud Drive / 微信文件传输 / 电脑** | 不要只存在 iPad 本地，分散存储降低数据丢失风险 |

### 📥 如何恢复数据

1. 在家长设置页点击 **"📥 导入完整备份"**
2. 选择之前保存的 `.json` 备份文件
3. 确认两次后恢复数据

### 🔄 换 iPad 怎么办？

1. 在旧 iPad 上导出完整备份
2. 把备份文件传到新 iPad
3. 在新 iPad 上安装 "打卡积分" App
4. 进入家长设置 → 导入完整备份 → 选择备份文件

## 八、常见问题

### Q: 添加到主屏幕后，打开还是有地址栏？

**A:** 说明没有正确安装 PWA。请确认：
1. 是不是用 **Safari** 打开的？（不可以用微信/Chrome）
2. 是不是通过 **"添加到主屏幕"** 安装的？（不是"添加到书签"）
3. 网站的 HTTPS 是不是有效的？（GitHub Pages 自动提供，不需要额外操作）

### Q: 显示"无法连接"或白屏？

**A:** 
1. 检查 iPad 是否连接了网络（首次加载需要联网）
2. 确认代码推送是否成功
3. 如果之前能用，突然不行了：可能是 GitHub Pages 服务暂时故障，等几分钟再试

### Q: 离線后能做什么？

**A:** 离线可以：
- ✅ 查看今日挑战页面
- ✅ 查看之前的打卡记录
- ❌ 不能提交新数据（需要联网同步到 IndexedDB）
- ❌ 不能兑换奖励

### Q: PIN 码忘记了？

**A:** 需要重新设置。在当前版本中，请参考以下步骤：
1. 在电脑上打开浏览器的开发者工具
2. 在 Application → IndexedDB → CheckinPWA → appConfig 中手动清除 `parentPinHash` 字段
3. 重新打开 App，进入家长设置 → 设置新的 PIN 码

### Q: iPad 上的图标不清晰？

**A:** 图标已经准备了 192x192 和 512x512 两个尺寸，iPad 会自动选择最合适的。如果觉得模糊，可能是图标本身的设计问题。

### Q: 部署后 iPad 一直显示旧版本？

**A:** 
1. 首先确认电脑端已运行 `node scripts/bump-cache-version.js` 并推送代码
2. 在 iPad 上完全关闭 App（从后台划掉）
3. 重新打开，等待 1-2 秒
4. 如果还是旧版，打开 Safari 访问一次网站链接再关闭
5. 再试一次从主屏幕打开
6. **不要清除网站数据** — 这会清空所有打卡记录！
