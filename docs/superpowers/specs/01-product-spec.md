# 01 - 暑假任务打卡系统 · 产品规格

> 版本：V2.0 | 日期：2026-06-30 | 状态：待实现
>
> **V2 更新：** 以《小学生暑假学习运动打卡积分系统设计方案》为母版重写规则。
> 旧 V1 草案（通用 study +5、大运动 +10、小运动 +3、completionRate 60%/90%、连续 3/7/14/30 送抽奖机会）已废弃。

---

## 0. 系统定位

> **低摩擦习惯积分制，而不是学习质量评价系统。**

核心原则：

- 孩子白天自主打卡
- 家长晚上统一确认
- 完成即得分
- 不做质量分档
- 不做部分完成分
- 不倒扣积分
- 不公开比较两个孩子
- 兑换只扣当前积分余额，不影响累计总积分
- 积分用于正向强化学习节奏、运动、英语输入
- V1 重点是让孩子愿意每天打开、愿意完成主线学习任务，而不是把规则做成复杂游戏

---

## 1. 项目目标

为两个小学低年级孩子（小明、小红）设计一个暑假任务打卡系统。
孩子白天自主打卡 → 晚上家长确认结算 → 积分累积 → 兑换奖励。

## 2. 设备架构

```
小明 iPad ── 独立运行 PWA ── 只存小明数据
小红 iPad ── 独立运行 PWA ── 只存小红数据
```

- 每台 iPad 绑定一个孩子
- 内置家长模式（PIN 码进入）
- 无独立家长端、无多设备同步
- 通过 Safari"添加到主屏幕"作为类 App 使用
- 配置可导出/导入在设备间复用

## 3. 孩子端页面

### 3.1 今日挑战页（默认首页）

| 页面状态 | 触发条件 | 展示内容 |
|----------|----------|----------|
| 未开始 | 今天尚未访问 | 全部任务待完成，"开始挑战吧！" |
| 进行中 | 部分任务已打卡 | 已完成打勾变暗，未完成高亮，实时预计得分 |
| 全部完成 | 全部任务已打卡 | 庆祝动画 + "等待家长确认 ✨" |
| 已确认 | 家长已结算 | 显示今日得分，不可再打卡修改 |
| 休息日 | 当天无任务 | "今天休息日 🌴" |

**任务卡片展示内容：**
- 图标（emoji）
- 任务名称
- 建议时长
- 分值
- 已打卡后状态为"待确认"
- 家长确认后显示"已得分"

**熏听任务特殊处理：**
- 允许选择/输入时长：0 / 30 / 60 / 90 / 120 / 150 / 180 分钟
- 显示预计积分（根据 rulesConfig.listeningRules 计算）

**操作规则：**
- 点击未完成任务 → 标记完成（childChecked = true）
- 点击已完成任务 → 取消打卡（家长确认前）
- 家长确认后所有打卡锁定
- 顶部显示：等级、连续打卡天数、当前积分

### 3.2 奖励页（积分兑换）

```
┌──────────────────────────┐
│ ⭐ 256 积分               │
├──────────────────────────┤
│ [ 🎁 积分兑换 ]            │  ← 默认 Tab
│                          │
│ 抽奖 Tab 隐藏或显示"暂未开放" │
└──────────────────────────┘
```

**兑换列表按钮状态：**

| 条件 | 按钮显示 |
|------|----------|
| 积分足够 + 未达上限 | 🟢 兑换 |
| 积分不够 | 🔒 还差 XX 分 |
| 积分够但已达周期上限 | 🔒 已达上限 |
| 有 pending 未兑现 | 🟡 等待兑现中 |

**兑换确认弹窗：**
显示商品名、所需积分、当前余额、兑换后余额、取消/确认按钮。

**奖励按 tier 分组显示：**
- 小奖励（small）：30–60 分
- 中奖励（medium）：100–180 分
- 高奖励（high）：300–450 分
- 超级奖励（super）：800–1200 分

### 3.3 积分流水页

按时间倒序展示 pointRecord，每条显示：日期、类型标签、说明、变动额、余额。

## 4. 家长端页面（PIN 码进入）

### 4.1 今日确认页

V1 家长端只有两个核心操作：

| 操作 | 说明 | 数据效果 |
|------|------|----------|
| 确认 ✓ | 该任务得分 | parentStatus = approved，parentPoints = plannedPoints（或按 duration 规则计算） |
| 撤销 ✗ | 该任务不得分 | parentStatus = rejected，parentPoints = 0 |

**不在 V1 UI 中出现：**
- 优秀 / 一般 / 较差
- 部分完成（partial）
- 质量扣分
- 倒扣积分
- 逐项质量评分

**结算页显示：**
- 学习积分（rawStudyPoints / cappedStudyPoints）
- 运动积分（rawSportPoints / cappedSportPoints）
- 总积分（rawTotalPoints / cappedTotalPoints）
- 是否基础学习全完成（basicStudyCompleted）
- 是否有效学习日（effectiveDayReason）
- 连续天数
- 连续奖励
- 保护卡获得情况

**操作约束：**
- 点击"确认结算"前，所有确认项可随意修改
- 点击"确认结算"后，整日锁定，不可直接改
- 结算后修正走 `manualOverrideRecord` + `pointRecord: adjust`

### 4.2 积分流水页

同孩子端，但可看到所有类型记录，家长可在此发起 adjust。

### 4.3 奖励管理

管理奖励商品（增删改），查看兑换记录，标记 pending → fulfilled。

### 4.4 设置页

```
├─ 👤 孩子资料
├─ 📋 任务模板配置（增删改、启用/停用、星期配置、subCategory、taskType）
├─ ⚙️ rulesConfig 配置（积分上限、有效学习日、连续奖励、兑换限制、抽奖开关）
├─ 🎁 奖励商品配置
├─ 💾 导出完整备份
├─ 📥 导入完整备份（提示覆盖）
├─ 📤 导出配置模板（含 rulesConfig、taskTemplates、rewards）
├─ 🔑 修改 PIN 码
└─ ℹ️ 关于
```

## 5. 积分规则（V2 新规则）

### 5.1 学习任务积分

#### 基础学习任务（V1 默认）

| 任务 | taskId | 分值 | 建议时长 | 类型 |
|------|--------|:----:|:--------:|------|
| 语文晨读 | chineseMorningRead | 1 | 15 分钟 | 基础学习任务 |
| 英语晨读 | englishMorningRead | 1 | 15 分钟 | 基础学习任务 |
| 语文主线 | chineseMain | 10 | 40 分钟 | 核心学习任务 |
| 数学主线 | mathMain | 10 | 40 分钟 | 核心学习任务 |
| 英语精读 | englishIntensive | 5 | 30 分钟 | 基础学习任务 |

**基础学习任务满分：** 1 + 1 + 10 + 10 + 5 = **27 分**

**基础全完成奖励：** 5 个基础任务全部确认完成，额外 **+3 分**

**每日基础学习最高：** 27 + 3 = **30 分**

#### 每周轮换任务

| 任务 | taskId | 分值 | 建议时长 | 说明 |
|------|--------|:----:|:--------:|------|
| 英语泛读 | englishExtensive | 4 | 30 分钟 | 输入型，压力较低 |
| 英语复述 | englishRetell | 5 | 15 分钟 | 输出型，难度更高 |

**规则：**
- 同一天只安排一个轮换任务（二选一）
- 不建议同一天精读 + 泛读 + 复述全部叠加
- 轮换任务不作为基础全完成奖励的必要条件
- 轮换任务正常计分，但不影响基础任务全完成奖励判断

#### 自愿加分项

| 任务 | taskId | 分值 | 建议时长 | 说明 |
|------|--------|:----:|:--------:|------|
| 数学浅奥 | mathThinking | 5 | 20 分钟 | 完全自愿 |

**规则：**
- 完全自选，不做无惩罚，抗拒就跳过
- 不影响基础积分
- 不作为有效学习日的必要条件
- 每天最多计 1 次

> 浅奥是有余力时的挑战，不是基础学习责任。

#### 英语熏听（duration 类型任务）

| 当日熏听累计时长 | 积分 |
|:------------:|:----:|
| 0 分钟 | 0 |
| 1–60 分钟 | 1 |
| 61–90 分钟 | 2 |
| 91–120 分钟 | 3 |
| 121–150 分钟 | 4 |
| 151–180 分钟 | 5 |

**公式：** 60 分钟以内 1 分；超过 60 分钟后每增加 30 分钟多 1 分；每日最多 5 分。

**规则：**
- 熏听每日上限 5 分
- 熏听不应给过高分，避免刷分
- 熏听不作为有效学习日的必要条件
- 只完成熏听不能算有效学习日

#### 放松型任务

| 任务 | taskId | 分值 | 建议时长 | 说明 |
|------|--------|:----:|:--------:|------|
| 英语动画 | englishCartoon | 1 | 40 分钟 | 放松型输入，低分 |
| 亲子课外阅读 | parentReading | 2 | 20–30 分钟 | 陪伴型，适度鼓励 |

**规则：**
- 放松型任务可以计分
- 不影响有效学习日判断
- 不作为基础全完成奖励必要条件
- 不应赋予过高分值

### 5.2 运动任务积分

| 类型 | taskId | 分值 | 说明 |
|------|--------|:----:|------|
| 大运动 | bigSport | 10 | 轮滑、游泳、篮球、足球、户外运动 40–60 分钟，每天 1 个 |
| 小运动 1 | smallSport1 | 3 | 拉伸、跳绳、核心等 |
| 小运动 2 | smallSport2 | 3 | 开合跳、平衡练习等 |
| 小运动 3 | smallSport3 | 3 | 快走、引体向上等 |
| 小运动 4 | smallSport4 | 3 | 小燕飞、倒立、足弓运动等（默认关闭） |

**V1 小运动统一固定为 3 分，不需细分 2/3/4 分。**

**每日运动积分封顶：22 分。**

### 5.3 每日积分封顶

| 封顶项 | 上限 |
|--------|:----:|
| 学习积分封顶 | 45 分 |
| 运动积分封顶 | 22 分 |
| 总积分封顶 | 60 分 |
| 熏听积分封顶 | 5 分 |

**封顶逻辑：**
1. 先分别计算学习积分、运动积分
2. 学习积分超过 45 时截断为 45
3. 运动积分超过 22 时截断为 22
4. 学习 + 运动总和超过 60 时，最终入账积分截断为 60
5. 截断部分不入账
6. dailySummary 中记录 rawEarnedPoints 和 cappedEarnedPoints

### 5.4 有效学习日规则（废弃旧 completionRate >= 60% 规则）

**新的有效学习日使用任务组合判定。**

当天满足以下任一条件，即算有效学习日：

**条件 A：**
- 完成语文主线 chineseMain
- 完成数学主线 mathMain

**条件 B：**
- 完成语文主线 chineseMain 或 数学主线 mathMain 其中一个
- 完成英语精读 englishIntensive
- 完成任意晨读：chineseMorningRead 或 englishMorningRead

**无效情况：**
- 只完成晨读、熏听、动画：不算有效学习日
- 只完成一个主线任务：不算有效学习日，但正常得分
- 只完成运动任务：不算有效学习日
- 只完成放松型任务：不算有效学习日

**设计目的：** 有效学习日必须包含至少一个真正需要静坐投入的主线任务。
核心目标是鼓励孩子完成语文或数学主线，而不是靠轻任务刷连续天数。

**rulesConfig 表示：**
```js
effectiveDayRules: {
  mode: "taskCombination",
  validCombinations: [
    ["chineseMain", "mathMain"],
    ["chineseMain", "englishIntensive", "anyMorningRead"],
    ["mathMain", "englishIntensive", "anyMorningRead"]
  ]
}
```
其中 `anyMorningRead` 表示 chineseMorningRead 或 englishMorningRead 任一完成。

### 5.5 连续打卡奖励（废弃旧方案中附带的抽奖机会）

| 连续有效学习日 | 积分奖励 | 保护卡 |
|:------------:|:--------:|:------:|
| 3 天 | +5 | 0 |
| 7 天 | +15 | 1 |
| 14 天 | +30 | 0 |
| 30 天 | +80 | 0 |

**发放规则：**
- 连续奖励只在达到节点时发一次，不是每天重复发
- 每个连续周期内每个里程碑只发一次
- 断签后连续天数归零，重新连续达到同一里程碑时可再次触发
- 休息日不增加、不打断连续天数
- 使用 streakAwardRecord 防重复发放
- 唯一键：childId + streakPeriodStart + milestone

### 5.6 家长确认得分

| 家长判定 | 得分 | parentStatus |
|----------|------|--------------|
| 确认 | 全额 plannedPoints（或按 duration 规则计算） | approved |
| 撤销 | 0 | rejected |

V1 UI 不展示 partial。

### 5.7 日界线

```
dayCutoffHour = 4（凌晨 0:00–3:59 算前一天）
```

|------|:----:|:----:|

| 选择一次家庭小游戏 | 150 | 🎯 |
| 周末去一次小公园 / 游乐区 | 180 | 🛝 |

**高奖励（high）：**

| 奖励 | 积分 | icon |
|------|:----:|:----:|
| 买一个小玩具 | 300 | 🧸 |
| 去一次喜欢的运动场地 | 350 | 🏟️ |

**超级奖励（super）：**

| 奖励 | 积分 | icon |
|------|:----:|:----:|

| 特别亲子日活动 | 1000 | 🌟 |
| 主题外出（科技馆 / 动物园） | 1200 | 🦁 |

### 6.2 兑换频率控制

| 奖励类型 | 兑换频率 |
|----------|----------|
| 小奖励（small） | 每天最多 1 次 |
| 中奖励（medium） | 每周最多 2 次 |
| 高奖励（high） | 每月最多 2 次 |
| 超级奖励（super） | 家长确认即可，不强制限频 |

### 6.3 兑换规则

1. 积分余额 ≥ 商品价格 才允许兑换
2. 不允许负分
3. 积分不足时按钮显示"还差 XX 分"
4. 兑换立即扣减积分（pointRecord: redeem -N）
5. 生成 redeemRecord（status: pending）
6. 家长兑现后标记 fulfilled
7. 取消兑换 → status: cancelled + pointRecord: refund +N
8. 不直接删除兑换记录

## 8. 保护卡规则（V1 最小实现，预留字段）

| 规则 | 说明 |
|------|------|
| 每连续 7 天完成 | 获得 1 张保护卡 |
| 用途 | 生病、外出、特殊情况保留连续天数 |
| 每月最多使用 | 2 次 |

**数据结构预留：**

```js
  id, childId, time,
  type: "earn" | "use" | "adjust",
  source, cards: +1 / -1, balance,

}
```

## 9. 积分账户设计

| 名称 | 是否会减少 | 用途 |
|------|:----------:|------|
| 当前积分余额 currentBalance | 会 | 用于兑换奖励 |
| 累计总积分 lifetimeEarnedPoints | 不会 | 记录长期成长 |

- 当前积分余额 = 所有 pointRecord.points 汇总
- 累计总积分 = 历史所有正向已确认获得积分总和
- 兑换不影响累计总积分
- 不在 child 上存裸 totalPoints
- 可缓存 level，但积分余额必须从 pointRecord 汇总计算
- pointRecord 只追加，不修改，不删除

## 11. V1 功能边界

### V1 必做

1. 每台 iPad 绑定一个孩子
2. 每日任务列表
3. 孩子自主打卡 / 取消打卡
4. 家长晚上确认（确认 / 撤销，不展示 partial）
5. 固定任务固定积分
6. 基础学习任务全完成奖励（+3）
7. 熏听递增积分（duration 类型任务）
8. 当前积分余额
9. 累计总积分
10. 积分流水
11. 奖励兑换扣分（按 tier 限制频率）
12. 连续打卡奖励（纯积分 + 保护卡，不送抽奖机会）
13. 数据导出 / 导入备份
14. 配置模板导出 / 导入（含 rulesConfig、taskTemplates、rewards）

### V1 暂不做

1. 孩子排名
2. 复杂抽奖（lottery.enabled = false）
3. 质量评分
4. 部分完成分（partial）
5. 倒扣分
6. 太多徽章
7. 过细任务难度分档
8. 云端账号系统
9. 家长逐项评价学习质量
10. 每日宝箱 / 星星 / 成长地图 / 终局挑战的完整自动化逻辑
11. 抽奖系统
